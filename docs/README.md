# OctoAcme — Project Management Overview

This repository contains a set of living process documents used by the OctoAcme delivery teams. The files in docs/ provide the lightweight, repeatable practices we follow to initiate, plan, execute, release, and improve projects.

Key points — at a glance

- Purpose-driven: Every project starts with a Project One-pager (problem, goal, success metrics) to ensure clarity of intent.
- Roles & ownership: Projects name a Project Manager (PM) and Product Lead (PdM); developers, QA, and stakeholders have clear responsibilities.
- Iterative planning: Work is broken into small, shippable increments with acceptance criteria and a Definition of Done.
- Team cadence: Daily standups, weekly delivery syncs, and end-of-sprint demos or milestone reviews.
- Tracking & quality: Use a project board (Backlog → Ready → In Progress → In Review → QA → Done), CI for tests and security scans, and unit/integration tests.
- Risk & communication: Maintain a Risk Register; use weekly status templates and defined escalation paths (Team → PM → Product Lead → Sponsor).
- Releases: Follow pre-release checks (tests, release notes, rollback plan) and post-deploy verifications; document release notes.
- Continuous improvement: Run retrospectives after sprints/releases, convert action items into backlog issues, and measure improvement impact.
